## Story

**As a** admin do sistema,
**I want** que deal_notes e deal_files tenham organization_id, RLS correta (criador + admin/diretor), e que as notas sejam salvas de forma unificada,
**so that** o sistema tenha isolamento de dados correto entre organizacoes e nao haja dualidade de armazenamento de notas.

## Acceptance Criteria

1. `deal_notes` possui campo `organization_id` UUID NOT NULL FK → organizations
2. `deal_files` possui campo `organization_id` UUID NOT NULL FK → organizations
3. Dados existentes sao migrados: org_id preenchido a partir do deal pai (backfill)
4. Registros orfaos (deal_id nao existe em deals) sao limpos antes do backfill
5. RLS de deal_notes: SELECT/INSERT org-level, UPDATE/DELETE criador OU admin/diretor
6. RLS de deal_files: SELECT/INSERT org-level, UPDATE/DELETE criador OU admin/diretor
7. Indexes idx_deal_notes_org_id e idx_deal_files_org_id existem
8. DealDetailModal salva notas em deal_notes (nao mais como activities type=NOTE)
9. Soft delete implementado: contacts e deals usam `WHERE deleted_at IS NULL` em todas as queries
10. Funcao delete de contacts e deals faz `UPDATE SET deleted_at = NOW()` (nao DELETE fisico)
