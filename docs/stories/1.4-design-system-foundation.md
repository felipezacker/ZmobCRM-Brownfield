# Story 1.4: Design System Foundation -- Tokens, Componentes Base e Button Unificado

**Epic:** EPIC-TD (Resolucao de Debitos Tecnicos)
**Status:** Ready
**Sprint:** 1-2 (Semana 1-4)
**Estimativa:** ~52-86 horas
**Prioridade:** P0/P1
**Agente:** @dev (Dex)

---

## Descricao

O ZmobCRM possui taxa de adocao de design tokens de 0,47%, com 184 ocorrencias de 16 tokens shadcn inexistentes em 69 arquivos, 557 botoes `<button>` raw ad-hoc sem componente padrao, e 4 pares de componentes V1/V2 com 1.356 linhas de codigo morto. Esta story estabelece a foundation do design system: definir tokens semanticos, corrigir componentes base, remover codigo morto, criar Button unificado, e adicionar ErrorBoundary.

Segue as Phases 1 e 2 do plano de Brad Frost (Design System Architect).

## Criterios de Aceitacao

### AC-01: Tokens Semanticos Definidos
- **Dado** que 16 tokens shadcn sao referenciados mas nao existem em globals.css
- **Quando** os tokens forem definidos
- **Entao** todos os 6 componentes que dependem deles (Button, Card, Badge, Tabs, Alert, Popover) renderizam corretamente em light e dark mode (T-09)

### AC-02: Componentes Base Corrigidos
- **Dado** Card, Badge, Tabs, Alert, Popover usando tokens inexistentes
- **Quando** migrados para tokens validos
- **Entao** renderizam com cores corretas em ambos os modos (T-09)

### AC-03: Codigo Morto Removido
- **Dado** 4 pares de componentes V1/V2 com V2 tendo 0 importacoes
- **Quando** V2/Focus forem deletados
- **Entao** -1.356 linhas removidas E nenhum import quebrado E build passa (T-17)

### AC-04: Error Boundaries
- **Dado** zero Error Boundaries no codebase
- **Quando** implementados
- **Entao** existe 1 ErrorBoundary global + 1 por feature page (6 pages) E erro de runtime mostra fallback (nao tela branca) (T-11)

### AC-05: Notificacao Unificada
- **Dado** dois sistemas de notificacao (Toast: 51 refs, NotifStore: 10 refs)
- **Quando** adapter criado
- **Entao** todas as acoes de feedback usam mesmo sistema (T-14)

### AC-06: Button Unificado
- **Dado** componente Button morto (0 importacoes) e 557 `<button>` raw
- **Quando** Button unificado for criado e migracao iniciada
- **Entao** todas as 24 combinacoes (6 variants x 4 sizes) funcionam em light/dark (T-24) E migracao reduz `<button>` raw para <300
