## Descricao

O ZmobCRM possui funcoes de banco com SECURITY DEFINER que executam com privilegio maximo, ignorando RLS. As funcoes `mark_deal_won()`, `mark_deal_lost()` e `reopen_deal()` podem ser chamadas por qualquer usuario sem validacao de autorizacao. Alem disso, `get_dashboard_stats()` (versao sem parametro) vaza dados cross-tenant, e `handle_new_user()` aceita role via metadata, permitindo que um usuario se auto-promova a admin no cadastro -- invalidando TODA a seguranca RBAC.

### AC-02: get_dashboard_stats() Seguro
- **Dado** a versao antiga de `get_dashboard_stats()` sem parametro de organizacao
- **Quando** a migration for executada

- **Entao** `next build` completa sem erros E bundle de producao nao contem faker (T-19)

- **Entao** existe apenas uma fonte de verdade para configuracao Tailwind E build nao quebra (T-23)
