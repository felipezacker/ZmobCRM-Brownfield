## User Story

**Como** admin ou diretor de uma imobiliária,
**Quero** visualizar e trocar o corretor responsável por leads, contatos e deals diretamente na interface,
**Para que** eu possa redistribuir a carteira de clientes entre os corretores da equipe de forma ágil.

---

## Acceptance Criteria

- [x] AC1: Existe um componente reutilizável `<CorretorSelect>` que exibe um dropdown com todos os corretores da organização (query `profiles` WHERE `role IN ('admin','diretor','corretor')` AND `organization_id`)
- [x] AC2: Para usuários com role `admin` ou `diretor`, o componente renderiza como **dropdown editável**
- [x] AC3: Para usuários com role `corretor`, o componente renderiza como **texto read-only** mostrando o nome do corretor responsável
- [x] AC4: O `<CorretorSelect>` está integrado na **edição de contato** (ContactFormModal / ContactFormModalV2) com label "Corretor Responsável"
- [x] AC5: O `<CorretorSelect>` está integrado no **detalhe do deal** (DealDetailModal) com label "Corretor Responsável"
- [x] AC6: Ao **criar** um deal/contato como corretor, o `owner_id` é automaticamente preenchido com `auth.uid()` (comportamento atual mantido)
- [x] AC7: Ao **criar** um deal/contato como admin/diretor, o dropdown permite selecionar qualquer corretor (default: o próprio usuário)
- [x] AC8: RLS de UPDATE em `deals` e `contacts` impede que um corretor altere o campo `owner_id` (somente admin/diretor podem)
- [x] AC9: Typecheck (`npm run typecheck`) passa sem erros
- [x] AC10: Lint (`npm run lint`) passa sem erros

## Scope

### IN Scope
- Componente `<CorretorSelect>` reutilizável (dropdown + read-only mode)
- Integração em ContactFormModal, ContactFormModalV2, DealDetailModal e CreateDealModalV2
- Lógica de auto-assign na criação (corretor = criador por default)
- Dropdown para admin/diretor na criação com seleção de corretor
- Ajuste RLS de UPDATE para proteger `owner_id` contra alteração por corretor
- Hook `useOrganizationCorretores` para listar corretores da org

### OUT of Scope
- Cascata de reatribuição contato→deals (Story 1.4)
- Tela de cockpit (Story 1.5)
- Notificação ao corretor quando reatribuído
- Histórico de reatribuições (audit log)
- UI de leads (tabela `leads` existe no banco mas não há tela — será adicionado quando a tela for criada)

## Risks

| # | Risco | Probabilidade | Impacto | Mitigação |
|---|-------|---------------|---------|-----------|
| R1 | RLS de UPDATE restritivo pode quebrar edições legítimas do corretor em outros campos | Média | Alto | Policy deve restringir apenas alteração de `owner_id`, não de outros campos. Testar update de outros campos como corretor. |
| R2 | Policies de UPDATE podem conflitar com as existentes da Story 1.1 | Baixa | Médio | Verificar policies existentes e usar `CREATE OR REPLACE` ou `DROP + CREATE` |
| R3 | Performance do dropdown com muitos corretores | Baixa | Baixo | Query filtrada por org + role, índice já existe em profiles |

## QA Results

**Reviewer:** @qa (Quinn) | **Date:** 2026-02-20 | **Verdict:** CONCERNS

### Findings

| # | Severity | Category | Description |
|---|----------|----------|-------------|
| 1 | HIGH | security/rls | RLS WITH CHECK usa `owner_id = auth.uid() AND owner_id = auth.uid()` — ambas referem `NEW.owner_id` no PostgreSQL. Funciona na prática (corretor não pode reatribuir para outro), mas lógica deveria ser `NEW.owner_id = OLD.owner_id` para expressar a intenção corretamente. Recomendo corrigir antes do merge. |
| 2 | MEDIUM | code | `useOrganizationMembers` usa useState+useEffect manual em vez de TanStack Query. Perde cache/dedup. Tech debt. |
| 3 | LOW | code | `useActiveDealsCount` tem variável `query` não usada (dead code). |
| 4 | LOW | a11y | `ContactFormModal` useEffect missing `fetchCount` no dependency array. |

### Recommendation
Fix #1 (clarify RLS condition) before merge. Items #2-4 can be addressed as tech debt.

### Re-Review (2026-02-20)

**Verdict: PASS**

All 4 findings resolved:
- **#1 FIXED:** RLS WITH CHECK simplified to single `OR owner_id = auth.uid()` with clear comments explaining USING=OLD, WITH CHECK=NEW semantics. Correct.
- **#2 FIXED:** `useOrganizationMembers` migrated to `useQuery` with 5min staleTime. Cache/dedup working.
- **#3 FIXED:** Dead code `const query` removed.
- **#4 FIXED:** `fetchCount` wrapped in `useCallback([contactId])`, added to useEffect deps. Stable reference.
