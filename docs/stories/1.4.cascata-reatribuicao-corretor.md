## User Story

**Como** admin ou diretor,
**Quero** que ao trocar o corretor responsável de um contato, o sistema me pergunte se desejo reatribuir também os deals ativos daquele contato,
**Para que** eu possa transferir toda a carteira de um cliente de uma vez, sem precisar editar cada deal individualmente.

---

## Acceptance Criteria

- [x] AC1: Ao alterar o `owner_id` de um contato via `<CorretorSelect>`, se o contato possuir deals ativos (não fechados), exibir um **checkbox de confirmação**: "Reatribuir também os X deals ativos deste contato para [nome do novo corretor]?"
- [x] AC2: Se o checkbox for marcado, o sistema atualiza o `owner_id` de todos os deals onde `contact_id = contato` AND `is_won = false` AND `is_lost = false` AND `deleted_at IS NULL`
- [x] AC3: Se o checkbox for desmarcado (ou contato não tiver deals ativos), apenas o contato é atualizado
- [x] AC4: A contagem de deals ativos é exibida dinamicamente no texto do checkbox (ex: "3 deals ativos")
- [x] AC5: Se o contato não tiver nenhum deal ativo, o checkbox **não aparece**
- [x] AC6: A operação de cascata é executada em uma **única transação** (contato + deals atualizados juntos, ou nenhum)
- [x] AC7: Typecheck (`npm run typecheck`) passa sem erros
- [x] AC8: Lint (`npm run lint`) passa sem erros

---

## Scope

### IN Scope
- Checkbox de confirmação no formulário de contato ao trocar corretor
- Query para contar deals ativos do contato
- Batch update transacional (contato + deals)
- UI feedback (loading state durante o batch update)

### OUT of Scope
- Cascata de leads (leads não têm relação com deals)
- Notificação ao corretor antigo/novo sobre a reatribuição
- Histórico/audit log da reatribuição
- Desfazer (undo) da reatribuição
- Reatribuição no detalhe do deal (já coberto na Story 1.3 individualmente)

## Risks

| # | Risco | Probabilidade | Impacto | Mitigação |
|---|-------|---------------|---------|-----------|
| R1 | Supabase client não suporta transações multi-tabela nativamente | Média | Alto | Usar RPC (function) no Supabase para garantir atomicidade |
| R2 | Deals com stages avançadas sendo reatribuídos podem causar confusão | Baixa | Médio | Apenas reatribuir, não alterar stage. Informar no checkbox quantos deals serão afetados. |
