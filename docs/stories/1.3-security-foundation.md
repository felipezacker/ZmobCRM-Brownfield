**Epic:** EPIC-TD (Resolucao de Debitos Tecnicos)
**Status:** InProgress

**Estimativa:** ~88-128 horas
**Prioridade:** P0/P1
**Agente:** @dev (Dex)

## Descricao

Complemento da seguranca iniciada na Sprint 1. Esta story cobre: (1) politicas RLS nas ~12 tabelas restantes com acesso aberto, (2) criptografia de API keys de IA que estao em texto plano no banco, (3) storage policies restritivas para deal-files, (4) isolamento do admin client que bypassa RLS nas AI tools, e (5) correcao das funcoes cross-tenant restantes.

O risco cruzado RC-01 (admin client + RLS permissiva = exposicao total) e RC-05 (API keys + storage = cadeia de exfiltracao) sao tratados nesta story.

## Criterios de Aceitacao

### AC-01: RLS nas 12 Tabelas Restantes
- **Dado** ~12 tabelas (boards, board_stages, activities, deal_notes, deal_files, products, deal_items, tags, custom_field_definitions, system_notifications, lifecycle_stages) com RLS permissiva
- **Quando** as migrations forem executadas
- **Entao** cada tabela tem politica RLS restritiva baseada em organization_id, seguindo o padrao hierarquico (admin/diretor/corretor)

### AC-02: API Keys Criptografadas
- **Dado** API keys de IA em texto plano em `organization_settings` e `user_settings`
- **Quando** a migracao para vault for concluida
- **Entao** keys estao no Supabase Vault E RPC segura para acesso E codigo da aplicacao usa a nova RPC

### AC-03: Storage Policies
- **Dado** storage bucket `deal-files` com policies permissivas
- **Quando** policies restritivas forem aplicadas
- **Entao** usuario de org A nao acessa deal-files de org B (via deal ownership) (T-07)

### AC-04: Admin Client Isolado
- **Dado** que AI tools usam admin client (service role) que bypassa RLS
- **Quando** refatorado para usar Supabase client com RLS
- **Entao** AI tools respeitam RLS E funcionalidade de IA nao regride (T-20)

### AC-05: Funcoes Cross-Tenant Corrigidas
- **Dado** `get_contact_stage_counts()` e `log_audit_event()` sem filtro de org
- **Quando** corrigidos
- **Entao** retornam apenas dados da organizacao do usuario autenticado (T-08)
