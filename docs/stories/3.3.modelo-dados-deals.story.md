## Story

**As a** corretor imobiliario,
**I want** registrar o tipo de negocio (venda/locacao/permuta), data prevista de fechamento e visualizar a comissao estimada em cada deal,
**so that** eu possa gerenciar meu pipeline imobiliario com informacoes relevantes e ter visibilidade do meu potencial de ganho.

## Acceptance Criteria

1. `deals` possui novos campos: `deal_type` (VENDA/LOCACAO/PERMUTA, default VENDA), `expected_close_date` DATE, `commission_rate` NUMERIC (CHECK 0-100, nullable)
2. `profiles` possui novo campo: `commission_rate` NUMERIC (default 1.5, CHECK 0-100)
3. Index `idx_deals_type_org` existe para filtro por tipo de negocio
4. Index parcial `idx_deals_expected_close` existe para forecast (apenas deals abertos com data prevista)
5. Comissao estimada e calculada no app: `deal.value * ((deal.commission_rate ?? corretor.commission_rate ?? 1.5) / 100)`
6. CreateDealModal exibe select para `deal_type` (Venda, Locacao, Permuta)
7. DealDetailModal exibe `expected_close_date` (date picker) e `deal_type` (select)
8. Cockpit do deal exibe comissao estimada como campo read-only (formatado R$)
9. Admin/diretor pode editar `commission_rate` de corretores no perfil
10. COMMENT ON documenta a logica de fallback chain da comissao
