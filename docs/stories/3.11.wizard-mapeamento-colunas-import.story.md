## Story

**As a** corretor imobiliario,
**I want** mapear manualmente as colunas do meu CSV para os campos do CRM durante a importacao,
**so that** eu possa importar planilhas com qualquer formato de cabecalho sem depender de nomes pre-definidos.

## Acceptance Criteria

1. Ao selecionar um arquivo CSV, o modal avanca para um step de "Mapeamento de Colunas" antes de importar
2. O step de mapeamento exibe uma tabela com cada coluna do CSV como coluna da tabela
3. Cada coluna tem um dropdown no header com os campos do CRM disponiveis (Nome, E-mail, Telefone, CPF/CNPJ, Tipo, Classificacao, Temperatura, Status, Etapa, Origem, CEP, Cidade, Estado, Data Nascimento, Notas)
4. Cada dropdown inclui a opcao "Ignorar coluna" para colunas irrelevantes
5. O sistema pre-seleciona automaticamente campos que o auto-matching consegue detectar (via HEADER_SYNONYMS existente)
6. Preview de ate 5 linhas de dados abaixo de cada coluna para facilitar a identificacao
7. Nao permite selecionar o mesmo campo CRM em dois dropdowns diferentes (exceto "Ignorar")
8. Toggle "Ignorar cabecalho" — quando ativado, a primeira linha de dados e tratada como dados (nao como header)
9. Botoes "Voltar" e "Proximo" para navegar entre os steps do wizard
10. O step final (Confirmacao) mostra resumo dos campos mapeados + modo de duplicados antes de importar
11. O backend aceita o mapping manual como JSON `{columnIndex: crmField}` no payload do import
12. Funcoes de normalizacao existentes (normalizeClassification, normalizeTemperature, etc.) continuam sendo aplicadas nos valores importados
