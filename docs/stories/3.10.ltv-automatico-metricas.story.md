## Story

**As a** diretor de imobiliaria,
**I want** que o LTV do contato seja atualizado automaticamente ao ganhar um deal e ter um dashboard de metricas da minha equipe,
**so that** eu possa acompanhar a performance dos corretores, distribuicao de leads, e funil de conversao sem calculo manual.

## Acceptance Criteria

1. Ao marcar deal como won (`markAsWon`), o `contacts.total_value` e incrementado com `deal.value` automaticamente
2. Ao reverter won (se aplicavel), o total_value e decrementado
3. Dashboard de metricas acessivel em `/contacts/metrics`
4. Metrica: Contatos novos por periodo (semana/mes) com filtro de date range
5. Metrica: Contatos novos por source (WEBSITE, LINKEDIN, REFERRAL, MANUAL) — grafico de barras
6. Metrica: Funil de conversao por stage (LEAD → MQL → PROSPECT → CUSTOMER) — funnel chart
7. Metrica: Distribuicao por classificacao e temperatura — pie/donut charts
8. Metrica: Performance por corretor (contatos atribuidos, deals won, LTV gerado) — tabela ranking
9. Todos os dados filtrados por organization_id (RLS + explicit filter)
10. Link para dashboard acessivel da lista de contatos

**markAsWon existente (`lib/supabase/deals.ts`):**
```typescript
async markAsWon(dealId: string) {
  const { error } = await supabase
    .from('deals')
    .update({
      is_won: true,
      is_lost: false,
      closed_at: new Date().toISOString(),
      updated_at: new Date().toISOString(),
    })
    .eq('id', dealId);
  return { error };
}
```
→ Adicionar: buscar deal.value + deal.contact_id, incrementar contacts.total_value

### Decisao: Charts

**Recomendacao:** CSS puro para charts simples (barras, donut via conic-gradient). Evitar adicionar dependencia de chart library (recharts, chart.js) para 5 graficos simples. Se o resultado visual nao for satisfatorio, considerar `recharts` (ja comum em projetos Next.js).

### Riscos
- LTV pode ficar inconsistente se deals forem editados apos won — mitigar: recalcular LTV somando todos deals won

- Stages sao dinamicos (UUID) — o funil precisa mapear IDs para nomes legiveis

### Testing
