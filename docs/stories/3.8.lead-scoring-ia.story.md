## Story

**As a** corretor imobiliario,
**I want** ver uma pontuacao automatica (0-100) em cada contato baseada em engajamento, valor e atividade,
**so that** eu possa priorizar meus contatos mais quentes e focar energia nos leads com maior probabilidade de conversao.

## Acceptance Criteria

1. Cada contato possui um `lead_score` (0-100) calculado automaticamente
2. Score e atualizado ao criar/editar contato, ao ganhar/perder deal, e ao completar atividade
3. Fatores de score conforme formula deterministicaz:
   - Interacao recente (ultima interacao < 7 dias = +20, < 30 dias = +10)
   - Total value/LTV (> R$0 = +15)
   - Tempo no stage atual (< 30 dias = +10, > 90 dias = -10)
   - Atividades completadas (> 5 = +15, > 2 = +8)
   - Preferencias preenchidas (perfil completo = +10)
   - Deals ativos (> 0 = +20, multiplos = +5 bonus)
   - Temperatura manual (HOT = +10, COLD = -10)
4. Badge visual na lista de contatos: vermelho (0-30), amarelo (31-60), verde (61-100)
5. Badge visual no cockpit do contato com score numerico e barra de progresso
6. Score visivel na lista de contatos como coluna sortable
7. AI tool `getLeadScore` disponivel para o chat IA explicar o score e sugerir proxima acao
8. Score recalculado via server action (nao em tempo real no client)
9. Coluna `lead_score` indexada para ordenacao eficiente

### Formula de Score (referencia)

```
score = 0
// Interacao recente
if lastInteraction < 7 dias: score += 20
else if lastInteraction < 30 dias: score += 10
// LTV
if totalValue > 0: score += 15
// Tempo no stage
if stageAge < 30 dias: score += 10
else if stageAge > 90 dias: score -= 10
// Atividades
if completedActivities > 5: score += 15
else if completedActivities > 2: score += 8
// Preferencias
if hasPreferences (property_types preenchido): score += 10
// Deals ativos
if activeDeals > 0: score += 20
if activeDeals > 1: score += 5
// Temperatura
if temperature == HOT: score += 10
else if temperature == COLD: score -= 10

return Math.max(0, Math.min(100, score))
```

### Riscos
- Score pode ficar desatualizado se recalculo falhar — mitigar com batch job
- Performance do batch recalculation com muitos contatos — limitar a 500 por batch
- Contatos antigos sem atividades terao score baixo — esperado (incentiva interacao)

### Testing
- **Foco:** Formula correta, triggers funcionam, UI badges corretas
- **Edge cases:** Contato sem deals/atividades (score = 0), contato com dados maximos (~100), temperatura COLD com muitos deals (score misto)
