# Story 1.6: Qualidade de Codigo -- TypeScript Strict, Testes e Observabilidade

**Epic:** EPIC-TD (Resolucao de Debitos Tecnicos)
**Status:** Draft
**Sprint:** 3-4 + Continuo
**Estimativa:** ~170-296 horas
**Prioridade:** P0/P1
**Agente:** @dev (Dex)

---

## Descricao

O ZmobCRM opera com `strict: false` no tsconfig (apenas `strictNullChecks` habilitado), cobertura de testes de ~7%, uso extensivo de `as any`, e zero Error Boundaries ou monitoring (Sentry DSN existe no .env.example mas sem integracao). Esta story habilita TypeScript strict mode incrementalmente, melhora significativamente a cobertura de testes, e estabelece observabilidade basica.

## Debitos Cobertos

| ID | Debito | Horas |
|---|---|---|
| TD-001 | Habilitar strict mode incremental no tsconfig | 40-80h |
| TD-004 | Cobertura de testes de ~7% para >=30% (meta 3 meses) | 120-200h |
| TD-013 | Eliminar `as any` extensivos (depende de TD-001) | 16-24h |

## Criterios de Aceitacao

### AC-01: TypeScript Strict Mode
- **Dado** tsconfig com `strict: false`
- **Quando** habilitado incrementalmente
- **Entao** `tsc --strict --noEmit` completa sem erros (T-18) E nenhum novo `as any` introduzido

### AC-02: Cobertura de Testes >=30%
- **Dado** cobertura de ~7% (~29 arquivos de teste para ~404 de codigo)
- **Quando** novos testes adicionados
- **Entao** `npm test -- --coverage` mostra >=30% E todos os testes passam

### AC-03: Reducao de `as any`
- **Dado** uso extensivo de `as any` em tools.ts, contextos, route handlers
- **Quando** strict mode habilitado e tipos corrigidos
- **Entao** contagem de `as any` reduz em >=50% comparado ao baseline

## Escopo

### IN
- Habilitacao incremental de strict mode (arquivo por arquivo ou modulo por modulo)
- Testes unitarios para modulos criticos (seguranca, deals, contacts, leads, IA)
- Testes de integracao para fluxos principais
- Eliminacao de `as any` nos modulos ja migrados para strict
- Captura de baseline de metricas antes de iniciar

### OUT
- Cobertura de 50%+ (meta 6 meses -- continuo)
- Integracao Sentry/monitoring completa (Backlog -- TD-018)
- Rate limiting (Backlog -- TD-017)
